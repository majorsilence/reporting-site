SDK clients

Utilisez Majorsilence Reporting depuis n'importe quel langage

Chaque SDK encapsule le même moteur RDL — sous-processus, binaire AOT ou FFI native — pour générer des rapports depuis votre pile technique existante, sans dépendance .NET au niveau de l'appel.

Python majorsilence-reporting
SDK Python

Python 3.10+. Aucune dépendance runtime en dehors de la bibliothèque standard. Sous-processus, binaire AOT, ou FFI ctypes en processus.

sous-processus binaire AOT FFI ctypes

Rust majorsilence-reporting
SDK Rust

Édition Rust 2021. Une seule dépendance (libloading). FFI native uniquement — pas de sous-processus, pas de runtime .NET. Handle de bibliothèque thread-safe, adossé à Arc.

Tous les formats, tous les modes

Chaque SDK prend en charge pdf, xlsx, csv, xml, rtf, tif, et html dans tous les modes d'intégration. Le binaire AOT et les modes FFI natifs débloquent en plus xlsx_table, tifb, et mht. Les jeux de données en mémoire (sans base de données) sont disponibles dans tous les modes FFI natifs via add_data().
